## 버그

- [x]  경로 조회 시 하행 → 상행으로 조회 안됨  
       조회는 되는데 구간이 교대, 강남이고 교대, 남부터미널일 경우  
       강남 → 남부터미널 조회 시 교대, 강남으로 나옴  
  
  원인 : 등록된 구간이 교대 → 강남, 교대 → 남부이고, 경로를 찾아오긴한다.
  하지만 찾은 경로의 구간 정보를 그대로 찾은 결과이기 때문에
  강남 → 교대, 교대 → 남부가 아닌 등록된 구간 그대로 조회를 하게 되었음.
  이 결과로 Sections에서 getStations() 메서드가 처리하면서 남부 터미널은 지운 것.
  어차피 안지워도 순서가 달라서 안됨. 순서를 맞추려면 라이브러리에서 조회한 역정보를 던져야 하는데
  더 번거롭다고 생각함.

해결 : 역정보까지 같이 조회해서 Path에 역 정보도 상태로 가지게 함.
